# パチスロ・エンジン 実装状況および開発ロードマップ

## 1. システム概要
本プロジェクトは、HTML5 CanvasとTypeScriptを用いた、完全告知型パチスロ（ジャグラー風）のウェブブラウザ向けシミュレーションエンジンです。物理的なリールの描画・スクロール挙動と、フラグベースの内部抽選ロジックを完全に分離したMVC（Model-View-Controller）ベースのアーキテクチャを採用しています。

## 2. ディレクトリ構造と役割
| ディレクトリ / ファイル | 役割 |
| :--- | :--- |
| `src/main.ts` | エントリポイント。ゲームループ（requestAnimationFrame）の管理とキー入力（1, 2, 3, 0, Space）の監視。 |
| `src/state/GameState.ts` | ゲーム全体のグローバル状態管理（シングルトン）。内部フラグ（`hasBonusFlag`）、進行状態（`NORMAL` / `BONUS_STANDBY` / `BONUS_GAME`）、GOGO!ランプ要素を保持。 |
| `src/logic/ReelController.ts` | リールの回転制御、スナップ・滑り（現在固定2コマ）、停止位置の完全計算、および中段図柄とフラグを用いた入賞判定（`evaluateWin`）のコアロジック。 |
| `src/views/CanvasRenderer.ts` | Canvas API（2Dコンテキスト）を用いた純粋な描画処理。クリッピングマスクによるリール枠制御、テキスト座標計算、GOGO!ランプのネオングローエフェクト等。 |
| `src/constants/config.ts` | 実機に基づいた21コマのリール配列（`REEL_CONFIG`）、キャンバスやリールサイズの寸法定数、FPS制御パラメータ。 |
| `src/types/index.ts` | 個別図柄（`SymbolType`）、リール状態（`SPINNING` / `SLIDING` / `STOPPED`）、全体フェーズなどのシステム型定義。 |

## 3. リール制御仕様
### 3.1 リール挙動（ステートマシン）
- **SPINNING (通常回転)**: デルタタイム（dt）に基づき、毎フレームピクセル単位で等速にスクロール座標（`scrollY`）を加算。
- **SLIDING (滑り中)**: 停止ボタン押下時、現在のコマ境界から「+2コマ」先の物理座標を目標（`targetY`）として計算し、そこに到達（追い越し）する瞬間まで回転速度を維持して強制吸着（スナップ）させる。
- **STOPPED (停止)**: 座標が完全に固定され、配列上のインデックス（`topIndex` 等）が確定した状態。

### 3.2 無限スクロールロジック
リール帯（21コマ）を環状バッファとしてループ描画するため、リールの累積座標 `scrollY` に対してモジュロ演算 `scrollY % (SYMBOL_HEIGHT * REEL_LENGTH)` を行い、配列外参照を防ぐ堅牢な折り返しを実装済み。

## 4. ゲームフローと内部抽選規格
### 4.1 抽選と告知のフロー
1. **レバーON（Spaceキー）**:
   - `playState` が `NORMAL` の場合、1/5（20%）の確率で乱数抽選を実施。
   - 当選した場合は、内部的に `hasBonusFlag = true` をセット。
2. **リール停止（1, 2, 3キー）**:
   - ユーザー入力により、各リールが順次「滑り」を経て停止。
   - 全てのリールが停止した第3停止の瞬間に `evaluateWin()` メソッドを実行。
3. **告知演出（後告知）**:
   - リール上の図柄に関わらず、`hasBonusFlag` が `true` の場合はGOGO!ランプを点灯（ピンク発光）させ、状態を `BONUS_STANDBY`（ボーナス確定状態）へ移行。
4. **ボーナス入賞待ち**:
   - `BONUS_STANDBY` 滞在中は、次ゲーム以降もランプが点灯し続ける。
   - この状態で目押しにより中段に「7」または「BAR」が揃った時、フラグを消費しGOGO!ランプを消灯。コンソールに「🎉 ボーナス開始！」を出力して `BONUS_GAME` 状態へ移行する。

## 5. デバッグ機能
- **[0] キー**: 強制ボーナスフラグ成立コマンド。押下後、そのゲームの第3停止で確実にGOGO!ランプがペカる。

## 6. 実装ロードマップ

AIのコンテキスト制限やバグ混入リスクを抑えるため、ロジックとビュー（描画）、機能ごとに細かくフェーズを分けて実装を進めることを推奨します。

### ✅ フェーズ0：コアエンジンの基礎構築（完了済）
- [x] **0.1 プロジェクト基盤**: Vite + Vanilla TypeScript構成、MVCベースのディレクトリ構造の構築。
- [x] **0.2 ゲームループ**: `requestAnimationFrame` とデルタタイムを用いた高精度なFPS制御。
- [x] **0.3 リールのスピン・滑り制御**: キャンバス外参照を防ぐ環状座標計算（モジュロ演算）と、「2コマ先でピタッと止まる」吸着スナップロジック。
- [x] **0.4 描画エンジン**: Canvasの `clip()` メソッドを用いたリール枠のマスキングと、図柄のテキストレンダリング基盤。
- [x] **0.5 入賞判定・フラグ管理**: `GameState` によるグローバル状態管理と、「中段1ライン」での役合わせ判定。
- [x] **0.6 完全告知システム（本質ロジック）**: 内部抽選（乱数）と、成立フラグに基づく「第3停止でのGOGO!ランプ点灯（後告知）」およびネオン発光エフェクト。
- [x] **0.7 デバッグシステム**: [`0`キー]による強制ボーナスフラグ設定機能の実装。

### フェーズ3：リール制御の高度化「滑り」（ロジック層）
